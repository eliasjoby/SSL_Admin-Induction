# SSL Admin Induction 2026

## Overview

This repository contains administrative tasks and documentation for the 2026 SSL Systems Administration Induction. The modules cover system provisioning, security hardening, service deployment, and infrastructure automation.

## Core Objectives

- **System Infrastructure**: Provisioning and updating Ubuntu Virtual Machines.
- **Security Hardening**: Configuring restricted SSH access, UFW firewalls, fail2ban, and multi-factor authentication.
- **Access Control**: Implementing strict user permissions, space quotas, and data backups.
- **Web Services**: Deploying Nginx reverse proxies with strict routing and Content Security Policies.
- **Database Administration**: Securing local MariaDB deployments.
- **Networking**: Establishing secure remote access via WireGuard VPN.
- **Containerization**: Deploying and managing applications using Docker.
- **Automation**: Developing Ansible playbooks and roles for configuration management.

## Guidelines

- Prioritize initial security setup (Initial Setup, SSH, Firewall) to ensure a secure working environment.
- Follow the principle of least privilege for user, web service, and database interactions.
- Ensure all services and applications are properly isolated and secured against common vulnerabilities.

## Task Details

Detailed step-by-step procedures and configurations are divided into sequential task files (`task1.md` through `task9.md`).
